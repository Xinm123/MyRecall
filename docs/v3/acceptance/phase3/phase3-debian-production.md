## 1. 范围与目标

- 范围：Debian 生产形态部署与运维闭环验证，功能冻结（不新增业务功能）。
- 目标：完成服务化部署、可观测性、灰度升级与回滚能力收敛，并验证 7 天稳定运行。
- 对应 Gate 条件：
  - 7 天稳定运行。
  - Debian 部署脚本（systemd/容器）成功率 = 100%。
  - 回滚演练通过率 = 100%。
  - UI 关键路径在 7 天内致命中断次数 = 0。

## 3. 验收步骤

1. 在 Debian 完成部署（systemd/容器），验证服务启动、自恢复与依赖就绪。
2. 校验核心端点可用性与契约一致性（仅 `/v1/*`）。
3. 连续运行 7 天，采集 ingest/search/chat/UI 全链路日志与指标。
4. 每日执行健康检查：队列积压、错误码分布、延迟分位、资源占用。
5. 执行灰度升级演练，验证升级窗口内业务连续性与错误可见性。
6. 执行回滚演练，验证回滚后功能、数据与契约无破坏性偏差。
7. 运行 UI 关键路径脚本并统计 7 天致命中断次数。
8. 统计 Chat 引用覆盖率（non-blocking），若低于目标需提交整改动作与回归计划。
9. 汇总证据：原始日志路径、统计脚本/SQL、指标汇总表（P50/P95/P99）。
10. 输出 Gate 结论（Pass/Fail）与阻塞项。
