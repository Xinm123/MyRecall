## 0. screenpipe 对照与对齐

- screenpipe 做法：AX-first，OCR-fallback；对特定应用可配置 OCR 偏好；仅存储原始文本，不做索引时 AI 增强。
- 本阶段对齐：完全对齐 AX-first/OCR-fallback 行为与存储策略；以 Edge 作为唯一重处理节点；不生成 caption/keywords/fusion_text，且不写入 `ocr_text_embeddings`（014A/015A）。
- 口径约束（Scheme C SSOT）：AX 成功帧写 `accessibility`（不写 `ocr_text`），OCR fallback 帧写 `ocr_text`（不写 `accessibility`）；验收与 SQL 对账以 [data-model.md](../../data-model.md) §3.0.3 写入语义为准。
- P1 OCR 引擎策略：OCR fallback 固定为 RapidOCR（single-engine policy），本阶段不做多引擎切换/对比验收。

## 1. 范围与目标

- 范围：Edge 处理链路（AX-first + RapidOCR fallback）、按 `frames.text_source` 分表写入原始文本（AX→`accessibility`，OCR→`ocr_text`）、可追溯日志。
- 目标：验证决策正确性、可追溯性与 UI 上的处理来源可解释性。
- 对应 Gate 条件：
  - AX 成功帧写入 `accessibility` 表的正确率 = 100%（硬门槛，Fail-fast）。
  - AX-first/OCR-fallback 决策日志可追溯率 >= 95%。
  - 索引时零 AI 增强检查通过率 = 100%（caption/keywords/fusion_text/embedding 写入为 0）。
  - 处理来源字段 UI 展示完整率 = 100%。

### 2.1 S2b -> S3 数据流契约（Entry Contract，强制）

- Ownership 边界：S2b 仅负责采集并上传 `accessibility_text/content_hash/browser_url`；S3 独占 `text_source` 判定、OCR fallback、`accessibility/ocr_text` 写入职责。
- 输入契约：S3 必须消费 S2b 上传的原始 `accessibility_text`；该字段在 handoff 中必须始终存在且值为 string（允许 `""`，禁止 `null`）；`content_hash` key 也必须始终存在，但值可为 `null`。判定仅使用 `ax_text_normalized = TRIM(COALESCE(accessibility_text, ''))`。
- 阶段边界：S2b 的 `content_hash` coverage 仅验证 raw `accessibility_text` 非空样本是否带 hash，不以 `frames.text_source` 作为过滤条件。
- 上下文边界：S3 直接消费 S2b 已冻结的 `focused_context`（`app_name/window_name/browser_url`）与 `device_name`，不得在处理阶段重新拼装或覆盖这些字段。
- failure-class ownership：S3 负责 `AX empty / AX timeout / ocr_preferred_apps / OCR fallback success|failure / failed status` 的语义判定；不重新承担 permission capability 或 browser_url stale 的验收责任（见 [../../open_questions.md](../../open_questions.md) OQ-039）。
- 判定契约：
  - 命中 `ocr_preferred_apps` -> 强制 OCR（即使 AX 非空）。
  - 否则 `ax_text_normalized != ''` -> `text_source='accessibility'`。
  - 否则 -> OCR fallback；OCR 成功后 `text_source='ocr'`。
- 写入契约（Scheme C）：
  - `text_source='accessibility'` -> 必须存在 `accessibility(frame_id=frames.id)`，且不得存在 `ocr_text(frame_id=frames.id)`。
  - `text_source='ocr'` -> 必须存在 `ocr_text(frame_id=frames.id)`，且不得存在 paired_capture 写入的 `accessibility(frame_id=frames.id)`。
- 幂等契约：同一 `capture_id/frame_id` 的重试必须收敛到同一终态（child rows 不重复、不漂移）；发现终态冲突必须 fail-loud，不得静默修复。

## 3. 验收步骤

1. 准备三组样本数据：AX 可用、OCR 偏好、AX 缺失。
2. 触发处理流程并记录每条 capture 的决策路径（AX/OCR/fallback 原因）。
3. 检查分表写入与 `frames.text_source` 一致：
   - `text_source='ocr'` 的完成帧应有 `ocr_text` 记录；
   - `text_source='accessibility'` 的完成帧应有 `accessibility(frame_id)` 记录。
   - 单帧互斥：同一 `frame_id` 不得同时存在 `ocr_text` 与 paired_capture 写入的 `accessibility`。
4. 用 SQL 校验 AX 成功帧分表写入正确性（`missing_ax` 目标必须为 0）：
   ```sql
   SELECT COUNT(*) AS missing_ax
   FROM frames f
   LEFT JOIN accessibility a ON a.frame_id = f.id
   WHERE f.status='completed'
     AND f.text_source='accessibility'
     AND a.id IS NULL;
   ```
   判定规则：`missing_ax = 0` 为通过；任意非 0 直接判定本阶段 Gate `Fail`（Fail-fast）。
5. 用 SQL 校验 Scheme C 互斥不变量（Fail-fast）：
   - completed 帧不得双写：
     ```sql
     SELECT COUNT(*) AS dual_written
     FROM frames f
     WHERE f.status='completed'
       AND EXISTS (SELECT 1 FROM ocr_text ot WHERE ot.frame_id=f.id)
       AND EXISTS (SELECT 1 FROM accessibility a WHERE a.frame_id=f.id);
     ```
   - completed 帧不得双空：
     ```sql
     SELECT COUNT(*) AS no_child_rows
     FROM frames f
     WHERE f.status='completed'
       AND NOT EXISTS (SELECT 1 FROM ocr_text ot WHERE ot.frame_id=f.id)
       AND NOT EXISTS (SELECT 1 FROM accessibility a WHERE a.frame_id=f.id);
     ```
   - `text_source='ocr'` 不得缺失 `ocr_text`：
     ```sql
     SELECT COUNT(*) AS missing_ocr
     FROM frames f
     LEFT JOIN ocr_text ot ON ot.frame_id=f.id
     WHERE f.status='completed'
       AND f.text_source='ocr'
       AND ot.id IS NULL;
     ```
   判定规则：`dual_written=0 AND no_child_rows=0 AND missing_ocr=0`，任一非 0 直接判定本阶段 Gate `Fail`。
6. 用 SQL 与处理日志核对索引时增强禁用项：
   - `caption/keywords/fusion_text` 写入为 0；
   - P1 `ocr_text_embeddings` 表不应存在（`table_exists=0`）：
     ```sql
     SELECT COUNT(*) AS table_exists
     FROM sqlite_master
     WHERE type='table' AND name='ocr_text_embeddings';
     ```
   判定规则：`table_exists` 必须为 0；若非 0，判定为契约偏离并记录阻塞项。
7. 抽样核验至少 100 条处理记录，计算可追溯率。
8. 边界用例核验（每类至少 3 条样本）：
   - 空白 AX 文本（仅空格/换行）必须走 OCR fallback。
   - AX 超时但有非空部分文本必须标记为 `text_source='accessibility'`。
   - AX 超时且文本为空必须走 OCR fallback。
   - 命中 `ocr_preferred_apps` 时，即使 AX 非空也必须走 OCR 路径。
   - AX 不可用且 OCR 失败时，帧状态必须为 `failed`（不得误标 `accessibility`）。
9. 在 `/timeline` 内的 frame 详情视图（弹层/侧栏均可）检查处理来源字段与处理时间戳展示完整性（P1 不新增独立 `/frame/:id` 页面）。
10. 注入处理异常（OCR 超时/AX 为空）并验证错误码、重试与降级日志。

说明：若 P1-S2b 已将 Arc support 标记为 deferred，则后续阶段不得将 Arc 缺失视为 `browser_url` 主路径失败；Arc 仅作为 conditional browser evidence 处理。
