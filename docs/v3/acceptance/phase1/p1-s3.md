## 0. screenpipe 对照与对齐

- screenpipe 做法：AX-first，OCR-fallback；对特定应用可配置 OCR 偏好。
- 本阶段对齐：对齐 AX-first/OCR-fallback 行为；以 Edge 作为唯一重处理节点。

## 1. 范围与目标

- 范围：Edge 处理链路（AX-first + OCR-fallback + fusion）与可追溯日志。
- 目标：验证决策正确性、可追溯性与 UI 上的处理来源可解释性。
- 对应 Gate 条件：
  - AX-first/OCR-fallback 决策日志可追溯率 >= 95%。
  - 处理来源字段 UI 展示完整率 = 100%。

## 2. 环境与输入

- 运行环境：同 P1-S1。
- 配置与数据集：
  - 普通应用样本（AX 可用）
  - `ocr_preferred_apps` 样本（强制 OCR）
  - AX 不可用样本（触发 fallback）
- 依赖版本：OCR 引擎与处理 pipeline 版本

## 3. 验收步骤

1. 准备三组样本数据：AX 可用、OCR 偏好、AX 缺失。
2. 触发处理流程并记录每条 capture 的决策路径（AX/OCR/fallback 原因）。
3. 检查 fusion 文本产物存在且可被后续检索读取。
4. 抽样核验至少 100 条处理记录，计算可追溯率。
5. 在 frame 详情页检查处理来源字段与处理时间戳展示完整性。
6. 注入处理异常（OCR 超时/AX 为空）并验证错误码、重试与降级日志。
