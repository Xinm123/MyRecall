## 0.1 screenpipe 对照与对齐

- screenpipe 做法：AX-first，OCR-fallback；仅存储原始文本，不做索引时 AI 增强。
- 本阶段对齐：Scheme C 分表写入（AX 成功 -> `accessibility`；OCR fallback -> `ocr_text`），并保持 `frames.text_source` 可追踪。

## 1. 范围与目标

- 范围：Edge 处理链路（AX-first + OCR-fallback）、Scheme C 分表写入、决策日志与失败重试。
- 目标：验证分表写入正确性、决策可追溯性、索引时零 AI 增强、UI 可解释性。
- 对应 Gate 条件：
  - AX 成功帧写入 `accessibility` 表正确率 = 100%。
  - OCR fallback 帧写入 `ocr_text` 表正确率 = 100%。
  - AX-first/OCR-fallback 决策日志可追溯率 >= 95%。
  - 索引时零 AI 增强检查通过率 = 100%（`caption/keywords/fusion_text/ocr_text_embeddings` 写入为 0）。
  - 处理来源字段 UI 展示完整率 = 100%。

## 2. 环境与输入

- 运行环境：同 P1-S1。
- 配置与数据集：
  - AX 可用样本。
  - `ocr_preferred_apps` 样本（强制 OCR）。
  - AX 不可用样本（触发 fallback）。
- 依赖版本：OCR 引擎与处理 pipeline 版本。

## 3. 验收步骤

1. 准备三组样本（AX 可用、OCR 偏好、AX 缺失）并触发处理流程。
2. 记录每条 capture 的决策路径（AX/OCR/fallback 原因）。
3. 校验分表写入：
   - AX 成功样本必须写入 `accessibility`，且不写 `ocr_text`；
   - OCR fallback 样本必须写入 `ocr_text`，且不写 `accessibility`（paired_capture 路径）。
4. 校验 `frames.text_source` 与分表写入结果一致（`accessibility` 或 `ocr`）。
5. 用 SQL 与处理日志核对禁用项：`caption/keywords/fusion_text` 写入为 0，`ocr_text_embeddings` 新增记录为 0。
6. 抽样核验至少 100 条处理记录，统计可追溯率与分表正确率。
7. 在 frame 详情页检查处理来源字段与处理时间戳展示完整性。
8. 注入 OCR 超时/AX 为空场景，验证错误码、重试与降级日志。
