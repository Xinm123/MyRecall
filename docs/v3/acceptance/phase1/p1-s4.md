## 0. screenpipe 对照与对齐

- screenpipe 做法：`/search` + 过滤参数为核心，关键词路径 `search/keyword` 提供高性能检索。
- 本阶段对齐：线上检索仅 FTS+过滤，不引入 hybrid；对齐 API 语义与交互行为。

## 1. 范围与目标

- 范围：`/v1/search`、`/v1/search/keyword`、返回字段契约与 UI 过滤交互。
- 目标：确保检索准确、可过滤、可回溯，并验证 UI 与 API 参数映射一致。
- 对应 Gate 条件：
  - 精确词查询不低于对齐基线。
  - Search P95 <= 1.8s（标准时间窗）。
  - `/v1/search` 过滤参数契约完成率 = 100%。
  - 检索结果引用字段完整率 = 100%。
  - Search UI 过滤项契约映射完成率 = 100%。
  - 检索结果点击回溯成功率 >= 95%。

## 2. 环境与输入

- 运行环境：同 P1-S1。
- 配置与数据集：
  - 精确词查询集（>= 100 条）
  - 多过滤条件组合集（time/app/window/content_type）
- 依赖版本：search API 与索引版本

## 3. 验收步骤

1. 导入标准检索数据集，记录基线版本。
2. 运行精确词查询集，对比基线召回与排序。
3. 对 `/v1/search` 执行过滤组合测试（单参、多参、边界时间窗）。
4. 调用 `/v1/search/keyword` 验证关键词路径性能与正确性。
5. 抽样校验返回字段：`capture_id/frame_id/timestamp` 完整性。
6. 在 Search 页面逐项操作过滤器，抓包核对 UI 参数与 API 请求 1:1 对应。
7. 从检索结果点击回溯到 frame/timeline，统计成功率。
