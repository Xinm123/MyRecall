## 0. screenpipe 对照与对齐

- screenpipe 做法：事件驱动采集（app switch/click/typing/idle）并与上下文信息关联。
- 本阶段对齐：对齐事件驱动触发语义；保留 MyRecall Edge-Centric 拓扑。P1 触发枚举以 API/数据契约为准：`idle/app_switch/manual/click`；`window_focus` 不纳入 P1，若 P2+ 启用需按 screenpipe `capture_window_focus` 语义对齐（默认关闭）。

## 1. 范围与目标

- 范围：Host 事件驱动 capture、idle fallback、AX 文本采集并上传（主链路图片格式为 JPEG）。
- 目标：验证触发覆盖、性能与数据完整性，同时让 timeline 可见上传/入队状态。
- 对应 Gate 条件：
  - 切窗场景 95% capture 在 3 秒内入 Edge 队列。
  - 每分钟 300 次事件压测下 Host CPU < 25%，丢包率 < 0.3%。
  - 事件触发清单覆盖率 >= 95%。
  - 去抖 Gate：同 monitor 连续 `app_switch/click` 入库间隔 < `min_capture_interval_ms`（默认 200ms）违规数 = 0。
  - 去重 Gate：重复内容压测下可观测到 dedup skip，且 30s 保底写入成立。
  - 背压 Gate：触发通道 lag 时发生 collapse（折叠降级），无持续性 backlog 失控。
  - 新 capture 在 timeline 可见性通过率 >= 95%。

## 3. 验收步骤

1. 启动 Host/Edge，开启采集 debug 日志。
2. 按触发清单执行操作：
   - app switch（>= 50 次）
   - click（>= 100 次）
   - manual（>= 20 次）
   - idle fallback（>= 20 次）
3. 记录每次触发到入队的端到端延迟，统计 P95。
4. 执行 300 events/min 压测 10 分钟，记录 Host CPU 峰值与平均值。
5. 对账触发日志与入队记录，计算丢包率。
6. 抽样核验 AX 文本采集字段（app/window/timestamp/trigger）完整性。
7. 执行高频“内容不变”压测（app switch/click），确认出现 dedup skip 且每 30s 至少有一次保底写入。
8. 制造触发通道 lag（短时突发 > 处理能力），确认日志出现 collapse 事件且队列深度回落，无持续增长。
9. 执行 SQL 校验：
   - 去抖违规数（目标 0，同 monitor 以 `device_name` 分区）：
     ```sql
     WITH ordered AS (
       SELECT id, device_name, capture_trigger, timestamp,
              LAG(timestamp) OVER (
                PARTITION BY device_name
                ORDER BY timestamp
              ) AS prev_ts
       FROM frames
       WHERE capture_trigger IN ('app_switch', 'click')
     )
     SELECT COUNT(*) AS debounce_violations
     FROM ordered
     WHERE prev_ts IS NOT NULL
       AND ((julianday(timestamp) - julianday(prev_ts)) * 86400000.0) < 200;
     ```
   - 30s 保底写入（目标成立）：
     ```sql
     WITH samples AS (
       SELECT timestamp
       FROM frames
       WHERE capture_trigger IN ('app_switch', 'click')
         AND timestamp >= datetime('now', '-10 minutes')
     ),
     ordered AS (
       SELECT timestamp,
              LAG(timestamp) OVER (ORDER BY timestamp) AS prev_ts
       FROM samples
     )
     SELECT
       (SELECT COUNT(*) FROM samples) AS writes,
       COALESCE(MAX((julianday(timestamp) - julianday(prev_ts)) * 86400.0), 0) AS max_gap_sec
     FROM ordered
     WHERE prev_ts IS NOT NULL;
     ```
     判定：在“内容不变”压测窗口内，`writes` > 0 且 `max_gap_sec <= 30`（附原始明细）。
10. 抽样校验本阶段新 capture：`frames.snapshot_path` 指向 JPEG 文件（`.jpg`/`.jpeg`），并通过 `GET /v1/frames/:frame_id` 返回 `Content-Type: image/jpeg`。
11. 打开 timeline，确认新 capture 的“上传中/已入队/失败重试”状态可见，且 dedup 场景下时间线无长时空洞。
