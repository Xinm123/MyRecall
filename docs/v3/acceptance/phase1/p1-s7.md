## 3. 验收步骤

1. 逐项确认 S1~S6 前置 Gate 结果均为 Pass。
2. 执行关键路径脚本：`timeline -> search -> chat -> citation -> frame`。
3. 统计 TTS（capture 到可检索）P95。
4. 运行 chat 问题集，统计基于 deep link 的引用覆盖率并抽检真实性（`myrecall://frame/{id}` 与 `myrecall://timeline?timestamp=...`，ID/时间戳不得伪造）；同时验证引用点击后统一落到 `/timeline`（`frame` 链接通过 `/v1/frames/:frame_id/metadata`（timestamp resolver，最小稳定契约）解析 timestamp 定位）。评估 DA-8=A 效果，若覆盖率 < 92% 需产出 DA-8 B 阶段实现方案。
5. 执行故障注入场景，验证链路可恢复与错误可见。
6. 汇总功能清单、契约清单、异常场景清单完成度。
7. 归档证据：日志、截图、录屏、指标导出。
