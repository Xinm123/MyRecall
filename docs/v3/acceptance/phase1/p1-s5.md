## 0. screenpipe 对照与对齐

- screenpipe 做法：chat 通过搜索工具获取上下文，并在回答中体现可追溯来源。
- 本阶段对齐：对齐工具驱动检索与引用回溯能力；坚持 Edge 侧编排。

## 1. 范围与目标

- 范围：Chat orchestrator 的 grounding、工具调用与引用链路。
- 目标：确保“有依据回答”成为默认路径，引用在 UI 可点击回溯。
- 对应 Gate 条件：
  - Chat 引用覆盖率 >= 85%。
  - 无引用回答可被网关拒绝或标记。
  - Chat 工具能力清单完成率 = 100%。
  - Chat 引用点击回溯成功率 >= 95%。

## 2. 环境与输入

- 运行环境：同 P1-S1。
- 配置与数据集：
  - grounding 问题集（>= 80 条，含时间窗/应用过滤）
  - 不可回答问题集（>= 20 条）
- 依赖版本：chat orchestrator 与 tool schema 版本

## 3. 验收步骤

1. 运行 grounding 问题集，记录每轮回答与引用字段。
2. 统计引用覆盖率，抽检引用对应的 frame/timestamp 是否真实存在。
3. 对不可回答问题集验证：
   - 无依据时应拒答/标记低置信；
   - 不得生成伪造引用。
4. 检查工具调用覆盖：search/frame lookup/time range expansion 全部被触发验证。
5. 在 UI 中点击每条引用，验证可回溯到对应 frame/timeline。
6. 注入检索失败场景，验证错误可见性与可恢复行为。
