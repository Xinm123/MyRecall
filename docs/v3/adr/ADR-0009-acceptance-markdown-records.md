# ADR-0009 验收记录必须 Markdown 归档

- 状态：Accepted
- 日期：2026-02-26

## Context
- 用户明确要求：每个阶段（含子阶段）的验收都要用 Markdown 详细记录。
- 当前路线图虽有 Gate，但未强制定义验收记录归档规则与路径映射。

## Decision
- 对 Phase 与子阶段（P1-S1~S6）实行强制验收记录制度：
  - 每个阶段/子阶段在 Gate 判定前，必须先提交 Markdown 验收记录。
  - 验收记录未提交或内容不完整，视为 Gate 未通过。
- 记录归档路径统一为：`MyRecall/docs/v3/acceptance/`。
- 记录必须包含：目标范围、输入版本、测试环境、测试步骤、指标结果、结论（Pass/Fail）、风险与后续动作。

## screenpipe 参考与对齐
- screenpipe 工程内有明确测试与回归关注，但未强制采用当前这种“每个子阶段固定 Markdown 归档”的流程约束。
- 对齐结论：能力验证目标一致，可对齐；流程治理粒度更严格，属于 MyRecall-v3 的工程管理增强。

## Consequences
- 优点：
  - 验收可追溯，Gate 判定可审计，便于故障复盘与跨阶段对账。
  - 阶段质量基线更清晰，降低“口头通过”风险。
- 代价：
  - 文档维护成本增加，交付节奏会多一道文档门槛。

## Risks
- 若模板与字段定义不统一，记录质量会不稳定，反而增加沟通成本。

## Validation
- 在 P1-S1 起执行抽检：验收记录字段完整率必须为 100%。
- P2/P3 Gate 评审必须引用对应 Markdown 记录作为唯一验收依据之一。
